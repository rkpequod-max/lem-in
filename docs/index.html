🐜 Lem-in - Projet 42 School

Lem-in est un projet algorithmique qui consiste à faire passer un nombre donné de fourmis d'un point de départ à un point d'arrivée dans le minimum de tours, en évitant les collisions.

🏗️ Structures de Données

t_room - La Structure Centrale

typedef struct s_room {
    char *name;           // Nom de la salle (ex: "1", "A", "start")
    t_coord c;            // Coordonnées (x, y)
    int ant;              // ID de la fourmi actuellement dans la salle
    int start;            // 1 si c'est le point de départ
    int end;              // 1 si c'est le point d'arrivée
    t_list *lnks;         // Liste des salles connectées (voisins)
    struct s_room *f_to;  // Salle suivante dans le flux de fourmis
    struct s_room *f_fm;  // Salle précédente dans le flux
    struct s_room *next;  // Salle suivante dans la liste globale
} t_room;

t_bfs - Pour la Résolution

typedef struct s_bfs {
    int path_size;      // Longueur du chemin depuis le départ
    int send_ants;      // Nombre de fourmis à envoyer sur ce chemin
    int min_ants;       // Nombre optimal de fourmis calculé
    t_room **room;      // Pointeur vers le pointeur de salle
    struct s_bfs *prev; // Noeud précédent dans le chemin BFS
    struct s_bfs *next; // Noeud suivant dans la liste BFS
} t_bfs;

t_farm - Conteneur Principal

typedef struct s_farm {
    int ants;      // Nombre total de fourmis à déplacer
    t_room *rms;   // Pointeur vers la première salle
} t_farm;

🧮 Algorithmes

1. BFS (Breadth-First Search)

Le parcours en largeur permet de trouver le chemin le plus court entre le départ et l'arrivée.

  • Explore tous les voisins niveau par niveau
  • Garantit le chemin le plus court en nombre de salles
  • Utilise une file FIFO pour l'exploration

2. Gestion du Flux de Fourmis

Après chaque BFS réussi, on marque le chemin avec f_to et f_fm pour créer un "tunnel" virtuel où les fourmis circulent sans collision.

// Update Flow - Création des tunnels
r->f_fm = prv;    // Marque le retour
prv->f_to = r;    // Marque l'avant

3. Optimisation du Nombre de Tours

La fonction path_config() calcule le nombre optimal de fourmis par chemin pour minimiser le total de tours :

Formule : Tours = longest_path + num_ants - 1

⏱️ Complexité

  • BFS : O(V + E) où V = nombre de salles, E = nombre de liens
  • Simulation : O(tours × fourmis)
  • Mémoire : O(V) pour stocker le graphe
