📋 Vue d'Ensemble

Lem-in est un projet de la 42 School consistant à faire passer un nombre donné de fourmis d'un point de départ à un point d'arrivée dans le minimum de tours, en évitant les collisions.

Concept Clé : Le projet utilise un algorithme de BFS (Breadth-First Search) combiné à une gestion de flux de fourmis sur des chemins disjoints.

🏗️ Structures de Données

1. t_room (Structure Centrale)

typedef struct s_room {
    char *name;           // Nom de la salle (ex: "1", "A")
    t_coord c;            // Coordonnées (x, y)
    int ant;              // ID de la fourmi (0 si vide)
    int start;            // 1 si départ, 0 sinon
    int end;              // 1 si arrivée, 0 sinon
    t_list *lnks;         // Liste des voisins
    struct s_room *f_to;  // Salle suivante dans le flux (forward)
    struct s_room *f_fm;  // Salle précédente (from)
    struct s_room *next;  // Salle suivante dans la liste globale
} t_room;

Rôle critique : f_to et f_fm créent un "tunnel" virtuel pour le flux après le BFS.

2. t_bfs (Résolution)

typedef struct s_bfs {
    int path_size;      // Longueur du chemin
    int send_ants;      // Fourmis à envoyer
    int min_ants;       // Optimal calculé
    t_room **room;      // Double pointeur vers la salle
    struct s_bfs *prev; // Noeud précédent (backtracking)
    struct s_bfs *next; // Noeud suivant (file)
} t_bfs;

3. t_farm (Conteneur)

typedef struct s_farm {
    int ants;      // Nombre total de fourmis
    t_room *rms;   // Liste chaînée des salles
} t_farm;

🔍 Parsing et Validation

Le parsing se fait ligne par ligne via une machine à états finis (FSM) à 3 sections :

Section Attendu Fonction Clé
0Nombre de fourmisis_num()
1Salles (nom x y)is_room()
2Liens (A-B)is_link()
Règles strictes :
  • Un seul ##start et un seul ##end.
  • Noms de salles uniques, coordonnées uniques.
  • Pas de lien vers soi-même.

🕸️ Gestion du Graphe

Le graphe est représenté par une liste chaînée de salles, chaque salle contenant une liste chaînée de ses voisins (lnks).

Fonctions Principales :

  • push_room() : Ajoute une salle en tête de liste.
  • add_link() : Crée un lien bidirectionnel entre deux salles.
  • find_name() : Recherche une salle par son nom (retourne un double pointeur).

🧠 Algorithme BFS (Breadth-First Search)

C'est le cœur de la résolution. On cherche le chemin le plus court entre start et end.

Mécanique :

  1. Initialiser une file avec le noeud start.
  2. Tant que la file n'est pas vide :
    • Défiler le noeud actuel.
    • Pour chaque voisin non visité et non bloqué par un flux inverse :
    • Ajouter à la file et marquer le parent (prev).
    • Si voisin == end : Chemin trouvé !
  3. Backtracking via prev pour reconstruire le chemin.
Astuce Anti-Collision : Lors du BFS, on ignore les liens qui vont à l'encontre d'un flux existant (flow_to()) pour éviter les blocages.

🌊 Flux de Fourmis (Flow Network)

Une fois un chemin trouvé, on ne le garde pas juste comme une liste. On le transforme en tunnel.

Update Flow :

// Backtracking depuis end vers start
while (current != start) {
    current->f_fm = prev;  // Marque le retour
    prev->f_to = current;  // Marque l'avant (le tunnel)
    current = prev;
}

Cela permet aux fourmis de circuler sans collision car chaque salle sait exactement quelle est la suivante dans le flux.

⚡ Optimisation du Nombre de Tours

L'objectif est de minimiser : Longueur_Max_Chemin + Nombre_Fourmis - 1.

Stratégie :

La fonction path_config() teste différentes combinaisons de chemins :

  • Les chemins courts peuvent accueillir plus de fourmis.
  • Les chemins longs en accueillent moins (ils "bouchent" le pipeline).
  • On calcule le coût total pour chaque combinaison et on garde la meilleure.

🔄 Workflow Complet d'Exécution

  1. Parsing : Lecture et validation de la carte.
  2. BFS #1 : Trouve le chemin le plus court.
  3. Update Flow : Marque le tunnel f_to/f_fm.
  4. BFS #2 : Cherche un deuxième chemin disjoint (en respectant les tunnels existants).
  5. Path Config : Répartit optimalement les fourmis sur les chemins trouvés.
  6. Simulation :
    • Boucle tour par tour.
    • move_ants() : Avance toutes les fourmis dans leurs tunnels.
    • select_paths() : Injecte de nouvelles fourmis au départ si possible.
    • Affiche les mouvements L1-A L2-B.
